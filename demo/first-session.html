# supabase

## Purpose
REST access to our Supabase project's Postgres database through the PostgREST API, authenticated with the project's secret API key.

## Where it lives
The project base URL is the value of `SUPABASE_URL` (stored in `.env` at the project root, never in this module). The REST API is served under the base path:

```
{SUPABASE_URL}/rest/v1/
```

## Auth & access
Two secrets, declared by name in `module.yaml`, values read from `.env`:

- `SUPABASE_URL`: the Supabase project URL
- `SUPABASE_SECRET_KEY`: a secret API key; it bypasses Row Level Security, so it is server-side only and must never be exposed to a browser, logged, or written into module files

Every REST request must send the key in both headers:

```
apikey: <value of SUPABASE_SECRET_KEY>
Authorization: Bearer <value of SUPABASE_SECRET_KEY>
```

In scripts, read both values via `os.environ`; in shell, source `.env` first. Never hardcode the values.

## Key entities
- **Tables and views**: each exposed table or view in the `public` schema is available at `{SUPABASE_URL}/rest/v1/<table>`
- **Rows**: returned as JSON arrays of objects; selected, filtered, ordered, and paginated via query parameters
- **RPC functions**: exposed Postgres functions are callable at `{SUPABASE_URL}/rest/v1/rpc/<function>` with a JSON body of arguments

## Operations
- **Read rows**: `GET /rest/v1/<table>?select=<columns>&<filters>`
- **Filter**: `?<column>=eq.<value>` (also `gt.`, `gte.`, `lt.`, `lte.`, `neq.`, `like.`, `in.(a,b)`)
- **Order and paginate**: `?order=<column>.desc&limit=<n>&offset=<n>`
- **Insert**: `POST /rest/v1/<table>` with a JSON object (or array) body, plus `Content-Type: application/json`; add `Prefer: return=representation` to get the inserted rows back
- **Update**: `PATCH /rest/v1/<table>?<filter>` with a JSON body of changed columns; always include a filter
- **Delete**: `DELETE /rest/v1/<table>?<filter>`; always include a filter, an unfiltered DELETE affects every row

## Examples

Query a table with curl (loads secrets from `.env`, never inlines them):

```bash
set -a; source .env; set +a
curl "$SUPABASE_URL/rest/v1/todos?select=id,title,done&done=eq.false&order=id.desc&limit=10" \
  -H "apikey: $SUPABASE_SECRET_KEY" \
  -H "Authorization: Bearer $SUPABASE_SECRET_KEY"
```

Same query from Python (stdlib only, run with `uv run`):

```python
# uv run query_todos.py
import json
import os
import urllib.request

base = os.environ["SUPABASE_URL"].rstrip("/")
key = os.environ["SUPABASE_SECRET_KEY"]

req = urllib.request.Request(
    f"{base}/rest/v1/todos?select=id,title,done&done=eq.false&limit=10",
    headers={"apikey": key, "Authorization": f"Bearer {key}"},
)
with urllib.request.urlopen(req) as resp:
    print(json.dumps(json.load(resp), indent=2))
```

Insert a row:

```bash
set -a; source .env; set +a
curl -X POST "$SUPABASE_URL/rest/v1/todos" \
  -H "apikey: $SUPABASE_SECRET_KEY" \
  -H "Authorization: Bearer $SUPABASE_SECRET_KEY" \
  -H "Content-Type: application/json" \
  -H "Prefer: return=representation" \
  -d '{"title": "buy milk", "done": false}'
```
